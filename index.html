Step-by-Step Guide

Starting a New Thread to Continue the Same Project

⌄

Use this guide when your conversation gets long and you want to keep the same project moving without losing context.

Tip: If things feel slower or the logic starts drifting, it is usually the right time to start fresh with a clean thread and a clean context block.

BACKGROUND

⌄

Why Start a New Thread?

When a thread becomes too long, it may start to slow down the AI’s performance. To maintain a smooth workflow, it is recommended to start a new thread, which improves speed and keeps the conversation responsive.

Model’s Internal Algorithms

The algorithms used to process large amounts of text and maintain logical flow across long conversations can become inefficient as the input grows. At a certain point, the AI might not handle complex or long conversations as smoothly as shorter ones.

EXECUTION

⌄

1. Identify When to Start a New Thread

  • If you notice the conversation is slowing down or responses are taking longer than usual, it is a strong sign the thread is too long.
  • To avoid delays, start a new thread before performance is significantly affected.

2. Ask the AI to Provide Project Context

In your current thread, ask the AI to generate a project context block that summarizes the working state. 

I would like to start a new thread. Please provide the following project context block at the top for this project, so the new conversation can continue from where we stopped.
Copy Command - Request Project Context Block

3. Copy the Project Context Block

Once the AI provides the project context block, copy it. This block contains the key objectives, current state, and critical details so the new thread can continue seamlessly.

4. Start the New Thread

  • Open a new thread with the AI.
  • Paste the copied project context block at the very beginning of the new thread.
  • This is critical: it ensures the AI has the required background to continue without missing important details.

5. Continue Where You Left Off

After pasting the project context block, resume the project with your next instruction - just like you would in the original thread. 

Let’s continue from where we left off and pick up with the next steps for the project.
Copy Command - Continue the Project

6. Name the New Thread

For clarity, name the new thread something like "Project Name - Part 2". This helps you track project continuity across multiple threads.

SUMMARY

⌄

Quick Summary of Commands

To ask for the project context block

I would like to start a new thread. Please provide the following project context block at the top for this project, so the new conversation can continue from where we stopped.
Copy Command - Context Block

To continue the project in the new thread

Paste the context block into the new thread, then send: 

Let’s continue from where we left off and pick up with the next steps for the project.
Copy Command - Continue

Short Example

If you are writing a book called "The Lost Galaxy", your new thread should start with the context block provided by the AI. Then send the continue command. This keeps the AI focused on your plot and characters without the lag of an extremely long thread.
