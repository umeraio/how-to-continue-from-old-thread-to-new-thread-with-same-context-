Step-by-Step Guide: Starting a New Thread

Why Start a New Thread?

When a thread becomes too long, it may start to slow down the AI’s performance. To maintain a smooth workflow, it's recommended to start a new thread, which will improve speed and ensure the conversation remains responsive.

How to Start a New Thread and Continue the Same Project

1. Identify When to Start a New Thread

  • If you notice the conversation is slowing down or responses are taking longer than usual, it’s a good sign that the thread has become too long.
  • To avoid delays, it's best to start a new thread before the performance is significantly affected.

2. Ask the AI to Provide Project Context

In your current thread, simply ask the AI to provide a project context block.

Example command to use:

“I would like to start a new thread. Please provide the following project context block at the top for this project, so the new conversation can continue from where we stopped.”

This will ensure that the AI knows exactly what the project is about and where you left off.

3. Copy the Project Context Block

Once the AI provides the project context block, copy it. This block will contain important details about the project, including where you left off, the key objectives, and any relevant information that will allow the AI to pick up from where you stopped.

4. Start the New Thread

  • Open a new thread with the AI.
  • Paste the copied project context block at the very beginning of the new thread. This is critical as it ensures the AI has the necessary background to continue working on the project without missing any important details.

5. Continue Where You Left Off

After pasting the project context block, you can resume the conversation by asking questions or giving commands just like you would in the original thread.

Example follow-up message: "Let’s continue from where we left off and pick up with the next steps for the project."

6. Name the New Thread

For clarity, name the new thread something like "Project Name - Part 2" (or anything else that makes sense). This will help you keep track of the different parts of the project as it progresses across multiple threads.

Quick Summary of Commands:

To ask for the project context block:

“I would like to start a new thread. Please provide the following project context block at the top for this project, so the new conversation can continue from where we stopped.”

To continue the project in the new thread:

Paste the context block into the new thread and then continue with:

“Let’s continue from where we left off and pick up with the next steps for the project.”
