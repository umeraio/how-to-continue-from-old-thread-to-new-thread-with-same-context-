Step-by-Step Guide: Starting a New Thread

Why Start a New Thread?

When a thread becomes too long, it may start to slow down the AI’s performance. To maintain a smooth workflow, it's recommended to start a new thread, which will improve speed and ensure the conversation remains responsive.

Model’s Internal Algorithms

The algorithms used to process large amounts of text and maintain logical flow across long conversations can become inefficient as the input grows. At a certain point, the AI might not handle complex or long conversations as smoothly as shorter ones.

How to Start a New Thread and Continue the Same Project

1. Identify When to Start a New Thread

  • If you notice the conversation is slowing down or responses are taking longer than usual.
  • Best to start a new thread before performance is significantly affected.

2. Ask the AI to Provide Project Context

Ask for the context block in your current thread:

“I would like to start a new thread. Please provide the following project context block at the top for this project, so the new conversation can continue from where we stopped.”

3. Example: How to Use It

Step A: You type the command above in your old chat.

Step B: The AI replies with:
"Project: Web Design | Status: Part 1 Complete | Next: Coding Footer..."

Step C: You copy that block, open a New Thread, and paste it immediately.

4. Start the New Thread & Continue

Paste the block, then follow up with:

“Let’s continue from where we left off and pick up with the next steps for the project.”

Quick Summary of Commands:

Step 1: Get Context (Old Thread)

“I would like to start a new thread. Please provide the following project context block at the top for this project, so the new conversation can continue from where we stopped.”

Step 2: Resume (New Thread)

“Let’s continue from where we left off and pick up with the next steps for the project.”
